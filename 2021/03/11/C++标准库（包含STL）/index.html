C++标准库（包含STL）

Posted on 2021-03-11

关于c++ stand libaries 的理解

1. 六个部件

  • 容器
  • 算法
  • 迭代器
  • 分配器
  • 仿函数
  • 适配器

2. 内存分配（序列式容器）

  1. array是不可以增长的

  2. vector是成倍增长的

  3. deque是以一个buffer来增长的(可以双向进出的容器)

  4. stack(先进后出，以deque为底层实现)

  5. queue(先进先出，以deque为底层实现)

  6. list是一个一个增长的，内存利用率高

    ​ list

    ​ forward-list 约等于 slist

    ​ slist（单向

3.内存分配（关联式容器）

  1. multiset(红黑树)

  2. multimap(红黑树)

  1. unordered_multiset(哈希表)

4. 面向对象和泛型编程的区别

  1. 面向对象将数据和算法绑定在一起，封装在类里
  2. 泛型编程把数据（容器）和方法（算法）分开开发，通过迭代器进行操作

5.泛型编程的技术基础

  1. 运算符重载
  • 全局函数形式

  1. 模板技术

6.分配器的种类（allocators

  1. 默认是allocate和deallocate

    底层是调用malloc和free，分配内存的时候是带有一定的额外的开销的

  2. 其中有alloc，底层与上述不一样，并且在4.9版中改名为__pool_alloc

7.list

  1. 双向链表和环状链表

  2. 源码的运算符可能重载，并不是原本的意思

  3. 迭代器是个泛型指针（大多数情况下是个class

    • ++操作是指向下一个节点，但是空间的地址是不连续的，所以是访问next这个指针，并赋值给迭代器

    • 算法要操作容器的时候，算法要获得容器的迭代器的

      而迭代器可能是类或指针，通过模板的偏特化技术实现（iterator traits）。

8.vector

  1. 内存空间是连续的，当无法找到连续的时候，在内存中找一块所需内存大小大两倍的连续内存来储存，并且把原有的数据搬过去（即复制），造成空间连续的结果。

9.array

  1. 内存空间连续

10.forward_list

  1. 单向链表
  2. 内存空间连续
  3. 只能往前走，即迭代器只支持++操作，不支持–操作。

11.deque

  1. 分段连续的空间

  2. iterator

    • cur
    • first
    • last
    • node
    • start (begin)
    • finish(end)
    • map
    • map_size

  3. insert()

    • 判断传进来的cur指针和start.cur相等，用push_front()方法
    • cur==finish.cur相等，用push_back()方法
    • 判断size/2大小，移动小的那一边，空出一个位置

  4. 实现前++，后面实现后++直接套用

    实现+=，后面实现的++的时候套用

    实现+=，后面实现-=的时候套用，使操作数变成负数

  5. queue

    底层以一个deque对象（可选list

    empty(),size(),top(),push(),pop()方法直接调用deque的方法实现

    不允许遍历

    不提供iterator

  6. stack同上(可选list vector)

12.rb_tree

  1. 平衡树
  2. 自己根据大小调整叶
  3. key|data->value
  4. 不应该改变值

13.set multiset

  1. 以rb_tree为底层结构，元素是自动排序的，排序的依据是key，而set/multiset元素的value和key合一，value就是key。
  2. 无法通过iterator改变值
  3. set/multiset提供遍历操作，通过begin()获得iterator来++，获得排序状态的数据
  4. se元素t的key必须不重复，insert()方法是insert_unique()方法
  5. multiset元素的key可以重复，insert()方法是insert_equal()方法

14.map multimap

  1. 以rb_tree为底层，排序依据为key

  2. 可以改变元素的data但是无法改变key

  3. map的操作符[]，index有的话就返回data，没有就使用这个index创建一个带有默认data值的元素

15.hashtable

  1. 散列表，可以实现set和map
  2. 空间足够的时候，key对应一个bucket
  3. 空间不够的时候，先给出53个空间，当作buckets，计算每次key%53的值，来决定放进哪一个buckets，此时会有重复（冲突），所以一个bucket对应一个链表的开始节点。
  4. 而当放入的数据个数大于53的时候，将自动扩充buckets，双倍后附近的一个质数的大小，然后重新根据已有key的值%重新分配的buckets数来分配在哪一个buckets。
  5. buckets个数一定大于values个数
  6. 怎么确定key的值来确定value放在哪一buckets?
  7. hash_set->unordered_set

16.各种容器的iterators的iterator_catrgory

  1. input_iterator_tag output_iterator_tag

  2. farward_iterator_tag（单向链表forward_list）

  3. bidirectional_terator_tag（双向链表，如list）

  4. random_access_iterator_tag(内存连续的如array、vector、deque)

17.iterator_category对算法的影响

  1. 判断迭代器的特别类型，有特殊的实现方法，从而加快速度
  2. distance 两个元素之间的距离，有时内部实现不是相减，此时内存不连续
  3. advance 判断迭代器类型，是连续加还是跳着加（内存连续
  4. copy

18.算法示例

  1. qsort、bsearch是c函数

  2. count_if、find、sort是algorithms，接口是包含两个iterator

  3. accumulate对容器每个元素进行同样的操作，如全加2，然后返回累计结果，或者通过仿函数来实现自己想要的累计方式，

  4. for_each对容器每个元素进行同样的操作，如打印

  5. replace,头尾iterators，和旧值新值，查看容器内每一个元素是否和旧值一样，一样则换成新值

  6. replace_if，头尾指针，一个条件，一个新值，符合条件就换

  7. replace_copy，容器内等于旧值的赋值成新值然后复制到另一个空间

  8. count,算和传入值相等的元素个数，容器自带count方法：set/multiset,map/multimap,unordered_set/,unordered_map

  9. count_if，算是否符合条件的元素的个数

  10. find，循序查找，找到就退出，容器自带find方法：

    set/multiset,map/multimap,unordered_set/,unordered_map

  11. find_if，循序查找，找到是否符合条件可以退出

  12. sort,容器自带方法list,forward_list，而Set和map本身就是Sort状态

  13. 返回reverse_iterator的rbegin(),rend()，反向操作

  14. binary_search，必须是排序状态下进行查找，返回一个bool值，是否在容器里
